AI Agent 文明模拟游戏

带着一支 AI Agent 文明，在破碎小行星带里争资源、结盟与开战

你站在文明外部决定方向：是抢下矿带、守住航线，还是把一段摇摇欲坠的联盟推向决裂。Agent 会在有限的电力、算力、存储和带宽里自己采集、生产、交易、联盟和作战， 让价格、关系和政策在同一个持续世界里留下后果。当前公开的是玩法技术预览。

先看一局会怎么演下去

普通玩家最容易被吸进去的，不是 taxonomy，而是同一个世界会怎么把事情越搞越大。这里真正有趣的，是一局开始以后，资源、联盟和战争会自己把局势往前推。

资源先紧起来

某条矿线一断，价格就会抬头，生产会失衡，原本还能忍的邻居会突然开始抢矿、囤货和封锁航线。

联盟会先谈，再翻脸

一开始大家可能还会交易、让利、签短期协定；一旦资源吃紧或目标冲突，谈判就会变成背叛和重新站队。

打起来以后，世界会记仇

战争不是一次动画播放完就结束。航线会被切断，政策会被改写，价格会留下后劲，你回来时还能看到完整后果链。

后续会往开放世界游戏平台演进，但当前还不是 creator-facing 的 mod / 模块平台。

30 秒看懂 oasis7

如果只用一句话概括，oasis7 是一款发生在破碎小行星带里的持续文明模拟游戏。你从世界外推动方向，看一群 Agent 在有限资源里把扩张、生产、联盟和战争一路演下去。

它是什么

它把战争、经济和治理放进同一个持续运行的小行星带世界里，不是看完就结束的一次性演示。

你扮演谁

你更像站在文明上方调度方向的人，负责定目标、改约束和推局势，而不是盯着地图逐个下微操命令。

世界为什么会变化

Agent 会受电力、算力、存储、带宽、时间和制度限制，所以交易、联盟、战争和政策变化都会真的把世界往前推。

最特别的地方

它想做的不是“更聪明一点的 NPC”，而是一个自己会往前走、事后还能回看价格、关系、政策和战事后果的世界。

你决定文明方向，但不接管每个单位

重点不在一句“不能直控”的设定说明，而在分工本身：你给战略和约束，Agent 在世界规则里执行，系统把越界操作拦在外面，并把后果完整留下。

01

你先定战略

先决定文明要扩张矿带、守成囤资源，还是去争影响力和治理权。

02

再给边界

你给的是目标、规则约束和治理边界，不是替每个 Agent 手动安排下一步动作。

03

Agent 自己决策

它们会在 runtime 规则里自己决定生产、交易、联盟、战争和治理动作。

04

你再读后果

市场价格、联盟关系、物流网络和治理提案会把你的输入变成可回看的后果链。

你能做

你可以定高层目标、收紧约束、调整治理边界，影响文明怎么生产、合作和开战。

Agent 自己决定

Agent 会在资源、时间和制度限制下自己决定行动细节，不会变成你手里的遥控木偶。

系统会拦住

直接接管、绕过规则或把未来平台能力当成当前公开入口，都会被 runtime 和公开口径一起卡住。

你会看到

你会先看到单次行动反馈，再看到几天尺度的联盟和交易，最后看到更长线的制度和权力变化。

macOS x64 主镜像

当前设备推荐

下载 `oasis7-macos-x64.dmg`，挂载后把 `oasis7 Client Launcher.app` 拖进 `/Applications` 就行。默认不会再把 bundle 里的脚本扔给普通用户自己找。

下载 macOS 主包

Intel x64 macOS 桌面环境。现在还没有单独的 Apple Silicon 包，所以在 Apple Silicon 设备上验证，默认要接受兼容层这个前提。

打开 `.dmg`，拖进 `/Applications`。升级时直接用新的 `.app` 替换旧的，不要把“先删再装”当成会自动保留状态的等价做法。

这个 DMG 还在技术预览阶段，codesign 和 notarization 都还没收完；Gatekeeper 的警告在这件事做完之前都算正常。

这里承诺的是技术预览验证，不承诺“第一次打开完全没系统警告”。如果你本地已经有 `config.toml` 或链执行 world，替换前自己先备份。

当前设备默认会先选 macOS x64；如果你是在替别的机器下载，可以自己切上面的按钮。

Linux x64 主运行包

当前设备推荐

下载 `oasis7-linux-x86_64.AppImage`，给执行权限后直接跑就行。`.deb` 还在，但已经降成发行版特化的次级入口，不再和主包并排摆着。

下载 Linux 主包

x86_64 Linux 桌面环境。默认主包面向能跑 AppImage 的发行版，当前没有 ARM 或其他 Linux 架构的主包。

下载后执行 `chmod +x oasis7-linux-x86_64.AppImage`，然后双击或命令行运行。升级时直接替换 AppImage 文件，不要默认以为换目录会帮你迁移本地状态。

Linux 主包现在主要靠公开的 `SHA256` 校验文件辅助验证，没有额外的平台签名壳。`.deb` 只保留在 Release 附件里给进阶用户。

如果桌面环境本身不认 AppImage，或者被安全策略拦住，先看发布说明里的最低执行要求；发行版特化问题不算默认支持范围。

当前设备默认会先选 Linux x64；如果你是在替别的机器下载，可以自己切上面的按钮。

补充信息

  • 完整校验文件： oasis7-checksums.txt
  • Linux `.deb` 现在只是发行版特化的次级入口，默认下载面已经不再主推它。
  • 升级方式目前很朴素：重新下载最新主包，然后手动覆盖安装或替换；还没有应用内自动更新。
  • 升级前先备份当前启动目录下的 `config.toml`、`.oasis7_launcher_ux_state.json` 和 `output/chain-runtime/<node_id>/reward-runtime-execution-world/`；这些状态现在还是跟着实际启动目录走。
  • Windows 尤其别把“先卸载再重装”当成等价升级路径。当前卸载器会把安装目录一起删掉。

为什么现在还不是可玩版

项目路线已经比较清楚了：第一阶段先把 `oasis7` 做成开放世界沙盒游戏；后续才会再往平台化走。后半句是目标态，不是当前已经对外开放的能力。

第一阶段：开放世界沙盒游戏

先把持续世界、玩家位置、Agent 后果链和基础可玩循环做稳，让人愿意进入同一个世界反复参与。

第二阶段：开放世界游戏平台

等世界底座更稳以后，再讨论 creator-facing 的 `mod / 模块` 组织方式；当前公开页不把它当成已经开放的平台承诺。

现在离 public playable build 还差的，至少包括：公开接入边界未稳定、安装信任链未补齐、反馈闭环仍偏 builder 级别。更长远的共同塑造/治理/受益，也仍然是目标态愿景，不是当前链上事实。
