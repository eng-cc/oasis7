目标玩法设计的四条系统支柱

建议先走预览验证路径确认当前技术预览能否在你的机器上跑通；若需要深入调试，再进入完整构建与文档。以下为目标能力设计，不代表当前已向玩家开放。

三层循环统一驱动

设计目标是让短循环提供即时反馈，中循环提供阶段胜负，长循环沉淀长期策略成本。

目标价值：形成长期留存动机

战争 x 治理双线博弈

目标规则包含资源封锁、制度投票、区域条约等并行系统，提升策略层级。

目标价值：战斗之外仍有决策空间

玩家间接控制

目标交互是“定义目标与约束，由 Agent 执行”，减少机械操作并强调策略表达。

目标价值：提升指挥层决策感

高收益伴随高反噬

目标平衡是“高收益伴随高代价”，避免单一路径滚雪球并强化反制。

目标价值：胜利与代价同时成立

目标体验样例（设计中）

建议先走预览验证路径确认当前技术预览能否在你的机器上跑通；若需要深入调试，再进入完整构建与文档。这些样例用于说明目标节奏与设计方向，不代表当前可玩流程。

短循环目标（5~15 分钟）

目标是在短时间内给出清晰反馈，让玩家能快速完成一次策略确认。

中循环目标（1~3 天）

目标是让联盟、交易、冲突在阶段内形成可验证的策略差异。

长循环目标（数周）

目标是把治理提案与制度变化沉淀为长期世界规则。

本地技术验证 3 步（开发者）

离线验证（推荐）

先运行回放链路，确认数据产出、服务启动与 viewer 连接正常。

env -u RUSTC_WRAPPER cargo run -p oasis7 --bin oasis7_viewer_demo -- twin_region_bootstrap --out .data/world_viewer_data
env -u RUSTC_WRAPPER cargo run -p oasis7 --bin oasis7_viewer_server -- .data/world_viewer_data 127.0.0.1:5010
env -u RUSTC_WRAPPER cargo run -p oasis7_viewer -- 127.0.0.1:5010

在线验证（standard_3d / software_safe）

进入持续运行模式，验证实时事件流、观测面板与控制链路；硬件路径正常时走 `standard_3d`，弱图形环境下可降到 `software_safe`。

env -u RUSTC_WRAPPER cargo run -p oasis7 --bin oasis7_viewer_live -- llm_bootstrap --bind 127.0.0.1:5023 --web-bind 127.0.0.1:5011
# 仅诊断：--no-llm 只用于 observer/debug，不计入正式验证证据
env -u RUSTC_WRAPPER cargo run -p oasis7 --bin oasis7_viewer_live -- llm_bootstrap --no-llm --bind 127.0.0.1:5023 --web-bind 127.0.0.1:5011
env -u RUSTC_WRAPPER cargo run -p oasis7_viewer -- 127.0.0.1:5023

三步完成本地技术验证（首次编译约 5~15 分钟）

  1. 克隆仓库并进入目录。
  2. 执行 `env -u RUSTC_WRAPPER cargo check` 验证环境。
  3. 先跑回放确认链路，再切到在线模式验证持续运行。

参数说明、排障路径与测试矩阵请前往 文档中心（中文）。

当前技术预览公开的三种访问面为 `standard_3d`、`software_safe`、`pure_api`；其中 LLM / script 决策与 Local Provider lane 只描述执行方式，不构成额外公开模式。

首次构建完成后，后续重复启动通常可收敛到分钟级。

Linux x64 主运行包

当前设备推荐

下载 `oasis7-linux-x86_64.AppImage`，赋予执行权限后即可直接运行；`.deb` 已降级为发行版特化次级入口，不再和主包并列展示。

下载 Linux 主包

x86_64 Linux 桌面环境；默认主包面向可运行 AppImage 的发行版，当前不提供 ARM/Linux 其他架构主包。

下载后执行 `chmod +x oasis7-linux-x86_64.AppImage`，再双击或命令行运行；升级时直接替换现有 AppImage 文件，不要默认假设跨目录移动会自动迁移本地状态。

Linux 主包当前依赖公开 `SHA256` 校验文件辅助验证，不提供额外平台签名外壳；`.deb` 仅保留在 Release 附件中作为高级入口。

若桌面环境缺少 AppImage 集成或被安全策略拦截，请先看发布说明中的最小执行说明；发行版特化问题不进入默认支持路径。

已按当前设备优先选中 Linux x64；如果你要给其他机器下载，可切换上方平台按钮。

补充信息

  • 完整校验文件： oasis7-checksums.txt
  • Linux `.deb` 作为发行版特化次级入口，仅保留在 Release 附件与说明中，不再进入默认下载决策面。
  • 当前升级口径已固定为“重新下载最新主包并手动覆盖安装/替换”，还没有应用内自动更新。
  • 升级前请自行备份当前启动目录下的 `config.toml`、`.oasis7_launcher_ux_state.json` 与 `output/chain-runtime/<node_id>/reward-runtime-execution-world/`；这些状态当前仍按实际启动工作目录解析。
  • Windows 尤其要避免把“先卸载再重装”当作等价升级路径：当前卸载器会删除安装目录。
