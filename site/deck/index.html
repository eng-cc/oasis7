oasis7 / Roadshow Deck

一个会自己推进历史的 AI Agent 文明模拟游戏

世界发生在破碎小行星带里。你不直接点单位，而是在受控验证场景里站在文明外部设定目标、约束和治理边界， 然后看 Agent 如何把资源压力、交易、联盟和冲突一路推成会留下后果的历史。

题材 持续文明模拟，不是一次性 AI Demo
玩家位置 文明外部战略指挥者，不做逐单位微操
当前阶段 公开技术预览，已能验证世界后果链，尚不可误读成大众可玩版

这份 deck 负责顺序化讲述项目，不替代首页公开边界，也不替代正式规格真值。

01 / Why This Category Matters

为什么这不是一次性研究项目

Opportunity

机会不在“AI 更会说”，而在“世界终于能开始长历史”

如果 AI 只能做一次性聪明反应，它更像演示；如果 AI 能在一个持续世界里承担资源、制度和时间后果，它才有机会长成一个长期品类。

这件事值得继续看的三个原因

  1. 现在的模型和工具链，第一次让“自治世界 + 长期后果”开始有验证价值。
  2. 这不是单点 NPC 智能升级，而是把战争、经济和治理放进同一个持续系统里。
  3. 如果成立，它会更像一类新游戏底盘，而不是一波短期 AI 展示热潮。

02 / Agenda

这版只回答最关键的 5 个问题

接下来只讲五件事

  1. 它是什么游戏，而不是什么 AI 包装壳。
  2. 玩家反复在做什么判断，为什么这会好玩。
  3. 为什么这不是传统 4X / 经营 / 大战略的换皮。
  4. 现在已经证明了什么，哪些还没证明。
  5. 下一阶段唯一要拿下的 proof point 是什么。

03 / Problem

很多 AI 游戏 Demo 的问题，不是模型不够强，而是世界不够硬

A

会说话，但不承担后果

角色很会聊，但行为不受资源、时间、制度和真实状态约束，所以一离开脚本就塌。

B

看起来像世界，实际上像演示脚本

没有持续运行的时间线，也没有能被记录、检查、回放的状态变化。

C

玩家像在拉木偶线

玩家直接替系统做决策，世界本身没有自治性，于是长期策略乐趣也站不住。

`oasis7` 不是先追问“怎样让角色更会聊天”，而是先追问“怎样把世界做结实”。

04 / Thesis

`oasis7` 的主张：你玩的不是排产和点兵，而是改 incentives、改边界、押注文明会如何自组织

World-first

所有有效行为都要经过世界规则验证 不能跳过 runtime，不能靠后台改表把结果写死。

Persistent

世界持续运行，历史不会因为玩家离线而归零 组织、产线、联盟、冲突都在同一个时间轴上留下后果。

Player Leverage

玩家改的是激励、边界和联盟结构，不是逐单位操作 这让它和传统 4X / 经营 / 大战略的关键手感开始分叉。
目标不是做“更聪明一点的 NPC”，而是做一个会把玩家判断放大成文明后果的世界。

05 / Player Loop

玩家反复玩的，是一个高杠杆决策窗

01

扩张

抢矿、开新线、推物流，换更高增长，也换更高暴露。

02

保供

保住现有产线和航线，避免短期崩盘，但可能错失先手。

03

联盟

维持协定、让利或翻脸，换取中期安全、筹码或局势反转。

04

改治理

调整协议和规则，让下一轮 Agent 的选择空间整体偏移。

短期你看到价格与物流反馈，中期看到联盟与战争外溢，长期看到制度开始锁定未来局势。

06 / Why It Hooks

真正让人停不下来的，是你总要在“眼前止损”和“长期押注”之间选边

Representative Dilemma

矿带断供时，你要保本，还是保未来？

立刻撕毁协定，可以先保住自己阵营的产线；忍受短期亏损维持联盟信誉，可能换来下一轮更大的治理筹码。真正的快感不在动作本身，而在你知道这次判断会一路改写后面的扩张、风险和节奏。

短期反馈

价格、物流、产线稳定性几分钟内就会回到你脸上。

中期外溢

几轮后你会看到联盟关系、军事姿态和谈判筹码开始偏移。

长期锁定

治理规则和战争后果不会自动归零，它们会把下一阶段的选择空间一起改掉。

07 / Stack

技术底座存在的唯一原因，是让这套世界后果链真的站住

LLM Agents
WASM Modules
World Runtime
Consensus Layer
Distributed Storage & Networking
LLM 负责决策空间 但不直接充当世界真值。
WASM 承接高阶能力 把制度、产业、玩法规则做成可部署的模块。
Runtime + 共识负责真值 保证状态变化能被验证，而不是只停留在单机幻觉里。

08 / What Exists Today

现在已经去掉的风险，不是“能解释项目”，而是“世界后果链开始可验证”

已证明的事

  • 世界状态会持续推进，而不是每次都退回演示脚本。
  • 玩家输入已经能在受控验证里稳定改写价格、物流、联盟或治理后果。
  • 这些后果可以被 replay、live runtime 和审计轨迹回看。
  • 受控预览入口、文档和验证链路已经能承载对外讲解与验证。

当前边界

  • 这仍不是普通玩家可以直接进入的成熟产品。
  • 当前更接近“可研究、可验证、可对齐”的技术预览，而不是广泛可玩体验。
  • 这版 deck 的任务是讲清现状，不是把阶段边界包装掉。

`minimal / twin_region / triad_region` 片段属于受控验证场景回放摘录；更底层的验证入口仍在文档中心与测试手册。

公开口径必须继续保持：`technical preview / not playable yet`。

09 / Path

产品路径要按先后看：先把一个世界做成，再谈更广底盘

Phase 1

开放世界沙盒游戏

先把同一个持续世界里的参与感和后果感做实，让玩家愿意进入下一次判断窗。

  • 先让重复决策循环站住
  • 先让后果链可读且不靠扶着走
  • 技术预览向有限体验逐步收口

Phase 2

更广底盘（更后）

只有第一阶段被证明以后，才值得把规则集、模块生态和参与角色继续外扩。

  • 创作者参与可以再往前推
  • 规则和玩法可以继续演化
  • 但这不是当前要拿去证明的主轴

10 / Next Proof Point

下一阶段只看一个 proof point：玩家能否反复感到“一次判断会改写后续局势”

唯一 proof point 在 live preview 里，把一次供给、联盟或治理抉择稳定推成可感知后果，并让玩家愿意再做下一次判断。
近期去风险 把入口再缩短，把 replay 和审计证据讲清，把资源到政治到冲突的循环继续压稳。
风险边界 如果它只能在扶着走的案例里成立，或者玩家仍更像旁观者，这个 thesis 还没有被证明。
这个阶段不缺更大的愿景，缺的是一次更硬、能重复的体验证明。
