oasis7 / Roadshow Deck

一个会自己推进历史的 AI Agent 文明模拟游戏

世界发生在破碎小行星带里。玩家不直接接管单位，而是站在文明外部设定目标、约束和治理边界， 然后看 Agent 在资源压力下把扩张、生产、交易、联盟和战争一路推下去。

题材 持续文明模拟，不是一次性 AI Demo
玩家位置 文明外部战略指挥者，不做逐单位微操
当前阶段 公开技术预览，已能讲清与验证，尚不可误读成大众可玩版
这份 deck 负责顺序化讲述项目，不替代首页公开边界，也不替代 PRD / 项目文档真值。

01 / Agenda

为什么值得继续看

这套叙事会按 5 个问题往前走

  1. 它到底是什么游戏，而不是什么 AI 包装壳？
  2. 为什么这类世界有机会比普通 demo 更容易留住人？
  3. 玩家、Agent 和世界规则的分工到底怎么成立？
  4. 现在已经做到哪一步，哪些还没做到？
  5. 下一阶段要把什么先做实？

Deck Boundary

这里先解决顺序，不解决全部细节

官网和白皮书更适合自由浏览；deck 的价值在于，它能把问题、世界、产品边界和阶段判断按一条线讲清楚。

02 / Problem

很多 AI 游戏 Demo 的问题，不是模型不够强，而是世界不够硬

A

会说话，但不承担后果

角色很会聊，但行为不受资源、时间、制度和真实状态约束，所以一离开脚本就塌。

B

看起来像世界，实际上像演示脚本

没有持续运行的时间线，也没有能被记录、检查、回放的状态变化。

C

玩家像在拉木偶线

玩家直接替系统做决策，世界本身没有自治性，于是长期策略乐趣也站不住。

`oasis7` 不是先追问“怎样让角色更会聊天”，而是先追问“怎样把世界做结实”。

03 / Thesis

`oasis7` 的产品主张：先做世界，再让文明自己长出来

World-first

所有有效行为都要经过世界规则验证 不能跳过 runtime，不能靠后台改表把结果写死。

Persistent

世界持续运行，历史不会因为玩家离线而归零 组织、产线、联盟、冲突都在同一个时间轴上留下后果。

Emergence-first

不预设完整剧情，更关注规则会催生什么 组织、市场、协议和战争结果尽量从世界内行为里长出来。
目标不是做“更聪明一点的 NPC”，而是做一个能自己积累历史的世界。

04 / Player Role

玩家决定文明方向，但不接管每个单位

01

你先定战略

是扩张矿带、守成囤资源，还是去争治理权和协议解释权。

02

你给边界

你给的是目标、约束和治理规则，不是手动指定每一步操作。

03

Agent 自己执行

它们在资源、时间和制度限制下自己生产、交易、结盟、冲突。

04

世界把后果留下

价格、联盟、物流和治理提案会把你的输入变成可回看的后果链。

05 / Why It Hooks

这类游戏真正让人停不下来的，不是按钮，而是“局势会自己变脸”

资源先紧起来

某条矿线一断，价格会抬头，生产会失衡，原本还能忍的邻居会突然开始抢矿、囤货和封锁航线。

联盟会先谈，再翻脸

局势一开始可能只是交易和短期协定，但压力积累后，谈判会变成背叛和重新站队。

战争留下后劲

战争不是播完动画就结束。航线会被切断，政策会被改写，回来时还能看到完整后果链。

Core Feeling

从混乱长成秩序

这不是“我点了多少次操作”的快感，而是“我做的一次判断，把后面的扩张、风险和节奏全部改写了”的快感。

06 / Stack

技术底座不是炫技，而是为了让世界状态真的站住

LLM Agents
WASM Modules
World Runtime
Consensus Layer
Distributed Storage & Networking
LLM 负责决策空间 但不直接充当世界真值。
WASM 承接高阶能力 把制度、产业、玩法规则做成可部署的模块。
Runtime + 共识负责真值 保证状态变化能被验证，而不是只停留在单机幻觉里。

07 / What Exists Today

现在已经能研究、能验证，但还不能包装成公开可玩版

已经有

  • 官网首页与白皮书式总览，能把项目定位讲清楚
  • 文档中心、Viewer 手册、测试手册与公开 skill 入口
  • runtime / viewer / testing 的技术预览验证链路
  • GitHub Pages 静态发布与站点质量门禁

当前边界

  • 尚不可误读成普通玩家可直接进入的成熟产品
  • 当前更像可研究、可验证、可对齐的技术预览
  • deck 负责讲清现状，不负责把阶段边界包装掉
公开口径必须继续保持：`technical preview / not playable yet`。

08 / Path

产品路径分两段：先把一个世界做扎实，再把它做成底盘

Phase 1

开放世界沙盒游戏

先把参与感、探索感和成就感做实，让玩家愿意在同一个持续世界里待下来。

  • 玩法闭环与玩家动机先站住
  • 世界后果链与可读性先站住
  • 技术预览向有限体验逐步收口

Phase 2

开放世界游戏平台

再演进到可承载不同规则集、不同模块生态和不同参与角色的底盘。

  • 让创作者不只是消费者
  • 让规则与玩法可以继续演化
  • 但这一步还不是当前公开承诺

09 / Next Focus

下一阶段最重要的，不是再堆一层概念，而是把可体验闭环继续压实

玩法闭环 让“资源压力 -> 联盟变化 -> 战争后果”更稳定地变成玩家能反复感知的循环。
体验入口 把技术预览入口继续往“更易理解、更易验证、更不误导”收紧。
证据链 继续让公开叙事和 repo 真值对齐，避免一边宣传、一边靠文档补洞。
这个阶段最值钱的，不是把话说得更满，而是把“这个世界已经真的在动”证明得更硬。

10 / Close

一句话收口

`oasis7` 想做的不是“会聊天的角色集合”，而是一个在资源、时间、制度和共识约束下，真的会自己推进文明历史的世界。
