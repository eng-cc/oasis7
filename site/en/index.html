Four Target Design Pillars

Recommended order: verify the current technical preview path on your machine first; only then move into the full build/docs path if you need deeper inspection.These are target capabilities under development, not currently live player features.

Three Loops, One Strategic Arc

Target design: short loops for immediate response, mid loops for phase outcomes, long loops for long-horizon strategic cost.

Target value: sustained retention pressure

War x Governance Interlock

Target rules include blockades, policy votes, and regional treaties running in parallel.

Target value: strategic depth beyond combat

Indirect Player Control

Target interaction is intent-and-constraint control with agent execution under uncertainty.

Target value: command-level agency without click fatigue

High Reward, Real Blowback

Target balancing enforces trade-offs so upside always carries systemic counter-pressure.

Target value: every win has visible cost

Preview First: Local 3-Step Technical Verification

Offline Verification (Recommended)

Run replay first to verify data output, server startup, and viewer connectivity.

env -u RUSTC_WRAPPER cargo run -p oasis7 --bin oasis7_viewer_demo -- twin_region_bootstrap --out .data/world_viewer_data
env -u RUSTC_WRAPPER cargo run -p oasis7 --bin oasis7_viewer_server -- .data/world_viewer_data 127.0.0.1:5010
env -u RUSTC_WRAPPER cargo run -p oasis7_viewer -- 127.0.0.1:5010

Online Verification (`standard_3d` / `software_safe`)

Run continuously to verify real-time events, panel updates, and control flow; use `standard_3d` on the normal hardware path and fall back to `software_safe` on weak-graphics environments.

env -u RUSTC_WRAPPER cargo run -p oasis7 --bin oasis7_viewer_live -- llm_bootstrap --bind 127.0.0.1:5023 --web-bind 127.0.0.1:5011
# Diagnostics only: --no-llm is observer/debug-only and not formal verification evidence
env -u RUSTC_WRAPPER cargo run -p oasis7 --bin oasis7_viewer_live -- llm_bootstrap --no-llm --bind 127.0.0.1:5023 --web-bind 127.0.0.1:5011
env -u RUSTC_WRAPPER cargo run -p oasis7_viewer -- 127.0.0.1:5023

Three Steps for Local Technical Verification (First build: ~5-15 min)

  1. Clone the repository and enter the workspace.
  2. Run `env -u RUSTC_WRAPPER cargo check` to validate setup.
  3. Replay to verify closure, then switch to live mode for runtime checks.

Full parameters, troubleshooting, and test matrix live in the documentation hub (EN).

The current technical preview exposes three public access surfaces: `standard_3d`, `software_safe`, and `pure_api`. LLM/script decisions and Local Provider lanes describe execution style, not extra public modes.

After the first build, repeated local runs are usually ready in minutes.
