# 🎮 游戏体验反馈表

审计轮次: 5

## 标准字段（提交前必填 / 自动回填）
- 发布候选 / 版本号：用于横向对比，不可留空。
- 测试场景：如 `新手首玩` / `微循环复测` / `版本对比`。
- 测试者：记录执行者或来源（人测 / AI 辅助）。
- 会话时长：至少记录开始 / 结束时间或总时长。
- 关键操作链路：至少列出 1 条实际执行的主链路。
- 结论标签：`继续可玩` / `需观察` / `高优先级阻断` 三选一。
- 过程证据：截图、录屏、console、启动日志至少其一；高优问题必须有证据路径。
- 问题摘要：若存在负反馈，必须给出 1 句话摘要。
- 玩家杠杆摘要：必须明确回答“玩家做了什么、世界因此改变了什么、是否打开新决策”；不得只写“世界很活跃 / AI 很忙”。

## 评分口径
- 单题默认按 1~5 计分：最负向答案 = 1，最正向答案 = 5。
- 多选题（第 14 题）按体验信号计分：
  - 正向：`好奇` / `有成就感` / `有掌控感` / `想再试一次` 各 +1
  - 负向：`无聊` / `困惑` 各 -1
  - 中性：`紧张` 记 0
  - 第 14 题最终得分：`clamp(3 + 正向数 - 负向数, 1, 5)`
- 第 7 题“局势变坏原因”不直接计入总分，用于归因标签：
  - `系统随机` / `系统没听懂` => 归为 `system_control_gap`
  - `我策略有问题` => 归为 `player_strategy_gap`
  - `运气不好` => 归为 `variance`
  - `不确定` => 归为 `unclear`
- 维度分数：
  - 理解度 = Q1~Q3 平均分
  - 控制感 = Q4~Q6 平均分
  - 策略体验 = Q8~Q10 平均分
  - 可理解性 = Q11~Q12 平均分
  - 节奏与总体体验 = Q13~Q18（含 Q14 转换分）平均分
- 总评区间：
  - `4.2 ~ 5.0` => `继续可玩`
  - `3.2 ~ 4.19` => `需观察`
  - `< 3.2` => `高优先级阻断`
- 高优先级阻断附加规则：
  - 若 `控制感 <= 2.5` 或 `理解度 <= 2.5`，即使总评 > 3.2，也建议标为 `高优先级阻断`。
  - 若玩家明确勾选“不会继续玩”且开放题给出阻断性原因，也建议至少标为 `需观察` 并进入问题池。
- 玩家杠杆判定（独立于总评）：
  - 下列 5 项各记 `0/1`：`玩家关键动作明确`、`世界变化可归因到玩家`、`玩家能解释结果`、`结果打开新决策`、`这条链路会促使继续玩`
  - `player_leverage_score = 0~5`
  - `4~5` => `pass`
  - `2~3` => `watch`
  - `0~1` => `block`
  - 若只能证明“世界有活动 / AI 在运行”，但无法证明玩家让世界发生了什么变化，则必须记 `world_activity_only = yes`，且 `结论标签` 最高只能是 `需观察`。

感谢你参与测试！
