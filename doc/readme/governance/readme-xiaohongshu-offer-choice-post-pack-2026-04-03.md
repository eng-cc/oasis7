## 使用说明
- 这是第十篇，重点不是做择业万能公式，也不是劝大家“别去 AI”，而是把讨论收口到应届生第一份工作的真实优先级。
- 主语必须继续保持为“我 / 带过工程实习生的人 / 做项目的人”，不要切成职业导师口吻，也不要写成大厂招聘软广。
- 这篇的目标只有一个：
  - 讲清楚为什么在 `AI岗 vs 大厂后端` 这种看起来很典型的选择里，我会更看重平台、训练体系和工程基本功。
- 这篇必须保留的落点：
  - 这不是说 AI 不重要。
  - 恰恰相反，AI 是会持续渗进大厂和传统团队的大趋势。
  - 对应届生来说，第一份工作先获得平台、训练体系、带教和工程基本功，往往更重要。
  - 成熟团队里学到的不只是写代码，还有 review、稳定性、协作和线上问题处理。
  - 先进入一个能把人带起来的系统，再去追趋势，通常比一开始只押风口更稳。
- 不要滑向以下方向：
  - `AI岗不行 / 千万别去 AI`
  - `大厂一定更好` 这种绝对化结论
  - 岗位信息汇总、内推清单、薪资比较帖
  - 把账号说成招聘建议号或职业咨询号

## 正文建议
```text
昨天一个以前带过的实习生来问我 offer 怎么选。

一个是偏 AI 的岗位，
一个是传统大厂后端。

我最后给他的建议是：
如果你是应届生，我会更偏大厂后端。

不是因为 AI 不重要。
恰恰相反，我觉得 AI 这件事，大趋势已经挡不住了。

今天传统部门没做，
不代表下周、下个月不做，
说不定很快就全面转向了。

所以对应届生来说，
第一份工作我更看重的其实是：

平台
训练体系
带教
工程基本功

风口当然重要，
但第一份工作更重要的，
很多时候不是“你有没有站在最热的位置”，
而是“你有没有进一个能把你真正带起来的环境”。

尤其是工程岗。

在成熟团队里，
你学到的不只是写代码，
还有工程规范、review、协作、稳定性、线上问题怎么处理。

这些东西，后面会一直跟着你。

所以如果是我，
我会先选一个能把基本功打扎实的平台，
再去追趋势。

因为 AI 会越来越普及，
但扎实的工程能力，不会自己长出来。

如果是你，你会先选方向更热，还是平台更稳？
```

## 小红书发布前自检
- 标题要直接指向 `AI岗 / 大厂后端 / 第一份offer` 里的一个核心冲突，不要再叠“建议”“干货”这类弱词。
- 第一屏必须尽快说清楚：这不是反 AI，而是在讲应届生的优先级排序。
- 正文必须保留 `平台 / 训练体系 / 带教 / 工程基本功` 四个锚点，不要只剩泛化鸡汤。
- 不出现任何 `保offer`、`绝对正确`、`大厂最稳`、`AI岗别去` 这类绝对化话术。
- 结尾只保留一个互动问题，不追加简历修改、私信咨询或第二个 CTA。

## 小红书笔记分析结论
- 综合评分：`8.9/10`
- 标题类型：`冲突选择题 + 判断型`
- 风险等级：`🟡`
- 商业化程度：`7/10`
- 互动潜力判断：
  - `AI岗 vs 大厂后端` 本身就有天然评论区，容易触发真实站队。
  - 真正能撑住讨论的不是“去哪边更酷”，而是“对应届生第一份工作该优先拿什么能力”。
  - 这篇更偏评论和收藏双高，适合让读者代入自己现在的 offer 焦虑。
- 优先优化点：
  1. 不要把观点写成“一刀切”，保留“如果你是应届生”这个前提。
  2. 不要把 AI 写成对立面，重点放在“趋势会扩散，但底层训练不会自动补课”。
  3. 保留工程岗视角，不要发散成所有职业都适用的泛职业建议。

## 发布备注
- 这篇适合接在春招 / offer 焦虑语境里，但不要写成“我来帮你选工作”，更像“我昨天刚给一个实习生的真实建议”。
