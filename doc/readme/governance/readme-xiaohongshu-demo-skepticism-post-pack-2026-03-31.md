## 使用说明
- 这是第八篇，重点不是唱反调，也不是否定 demo 的价值，而是把视角换成“自己下场做过以后，更能看到本质了”。
- 主语必须继续保持为“我 / 做这个游戏的人 / 游戏工作室主理人”，不要切成行业评论员，也不要写成泛 AI 赛道观察。
- 这篇的目标只有一个：
  - 讲清楚为什么我现在看见很猛的 demo，会先问一句“这东西到底能不能扛”。
- 这篇必须保留的落点：
  - 不是不信 AI。
  - 也不是说 demo 没用。
  - 而是 demo 只负责高光，项目会把最差状态也摊出来。
  - 真正难的是放进项目后会不会露馅、会不会乱、成本扛不扛得住。
- 不要滑向以下方向：
  - 泛创业吐槽
  - “AI 都是泡沫”式逆风观点帖
  - 端着讲“产品化鸿沟”“长期一致性”这类抽象词
  - 暗示 `oasis7` 已经完整解决这类问题

## 正文建议
```text
做AI游戏以后，我有个很明显的变化。
现在看到那种特别炸的demo，我第一反应已经不是“哇，好牛”。
而是：这玩意儿能撑多久。

以前我真的很容易被打动。
一个角色接话接得很顺，一段演示跑得很丝滑，我就会觉得，完了，这方向要起飞了。

但真自己下场做之后，就慢慢不太敢这么想了。

因为demo这东西，太会挑自己状态最好的一分钟了。
项目不会。
项目会把它状态最差的时候也一起摊给你看。

你真往游戏里塞，就会开始遇到一堆很烦但躲不开的事：
它会不会前后说不一样的话，
换个场景还灵不灵，
玩家多玩几次会不会很快看穿，
出问题了能不能查，
最关键的是，成本扛不扛得住。

所以现在我不是不信AI。
我是越来越不敢只信“那一下很惊艳”。

能做出一个好看的demo，当然厉害。
但能把它真的放进游戏里，还不乱、不飘、不很快露馅，我现在觉得这才是真的难。

可能就是做久了，能看到本质了。
以前先看“猛不猛”。
现在先看“会不会很快塌”。

你们会更吃第一眼很猛的demo，还是更在意它能不能真的玩下去？
```
