## 正文建议
```text
最近有人问我，今年春招，如果站在游戏工作室主理人的角度，我会看什么样的人。

说实话，我现在不先看你会多少工具。
我先看，你能不能把事接住。

AI 起来以后，会一点代码、会一点设计、会一点内容的人越来越多。
所以现在真正稀缺的，反而不是“都会一点”的人。

我会更看这几种能力。

第一，能不能把自己做过的事讲清楚。
别只写“参与过项目”。
我更想知道，你具体做了什么，卡在哪，最后怎么收的。

第二，有没有真的做完过东西。
我不怕项目小，我怕全是半成品。
比起十个做了六成的作品，我更愿意看一个认真收过尾的。

第三，怎么用 AI。
“我会用 AI”已经不算亮点了。
我更在意的是，你是拿它代打，还是拿它提速，但最后自己判断、自己验收。

第四，有没有玩家感觉。
做游戏不是堆功能。
是你知不知道玩家会不会点，玩到哪里会烦，哪里会留下来。

第五，出问题时会不会往前推。
做项目一定会遇到需求变、方案废、版本卡。
这时候我更想看到的，不是“做不下去”，而是“我先给一个能落地的方案”。

所以如果你问我，今年春招我会看什么人。

不是最会包装自己的。
也不是简历写得最满的。
而是那个能把事情说清楚、做扎实、出问题还能收回来的人。

现在 AI 把很多表层能力拉平了。
反而让判断、执行、收口，变得更值钱。
```
