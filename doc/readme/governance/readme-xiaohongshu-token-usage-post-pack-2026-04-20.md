## 使用说明
- 这是第十四篇，重点不是晒成本吓人，也不是把 AI 写成已经能全自动取代团队。
- 主语必须继续保持为“我 / 我们这种长期做项目的人”，不要切成投资号、测评号或模型军备竞赛口吻。
- 这篇的目标只有一个：
  - 用真实累计账单让读者看懂，AI 一旦进入长期项目，最烧 token 的通常不是那几次看起来很猛的生成，而是后面持续发生的研发流程动作。
- 这篇必须保留的落点：
  - 截至本次统计，项目累计请求 `95,524`，累计 token `11780.14M`，累计花费 `$4680.42`。
  - `gpt-5.4` 是绝对主力：`59,943` 次请求、`7481.44M` token、`$3569.37`、占比 `63.5%`。
  - `gpt-5.3-codex` 是第二主力：`30,648` 次请求、`3818.50M` token、`$1051.96`、占比 `32.4%`。
  - 真正持续吃 token 的，往往是改需求、补文档、返工、修 bug、跑验证、回 review 和上下文切换。
  - 一旦项目周期拉长，token 就不只是“模型调用量”，而会变成一种真实的研发资源消耗。
  - 真正该优化的，不只是“少问一句”，而是减少无效上下文、重复沟通和低价值往返。
- 不要滑向以下方向：
  - `我们已经烧了很多钱，所以项目一定很厉害`
  - `哪个模型最贵就最值得追`
  - `这已经证明 AI 可以全自动做完整项目`
  - `这是融资能力 / 商业规模 / 用户规模证明`
  - 借题发挥暗示 `oasis7` 已上线、可玩、可投资

## 正文建议
```text
今天翻项目后台，
我自己都吓了一下。

这个项目到现在，
已经累计 95,524 次请求，
11780.14M Token，
也就是差不多 117.8 亿，
花了 $4680.42。

里面的大头其实很明显。

gpt-5.4 跑了 59,943 次，
吃掉 7481.44M Token，
花了 $3569.37，
占了 63.5%。

gpt-5.3-codex 跑了 30,648 次，
吃掉 3818.50M Token，
花了 $1051.96，
占了 32.4%。

剩下那些模型，
基本就是补位。

但我后来盯着这组数看了会儿，
发现重点其实不是“贵不贵”。

而是你把 AI 真放进一个长期项目里以后，
最烧 Token 的，
真的不是某一次看起来很猛的输出。

真正往里吃的，
是后面那些很日常的东西：

改需求，
补文档，
返工，
修 bug，
跑验证，
回 review，
还有不停地补上下文。

就是这些事，
一天天把 Token 慢慢吃掉了。

所以我现在的感觉挺直接的：
AI 一旦不只是拿来玩两下，
而是真的放进研发流程里，
它就很像团队里的一个正式成员。

只不过这个成员不拿月薪，
按 Token 计费。

也是因为这样，
我现在回头看这类账单，
已经不太会先想“怎么少问一句”了。

更像是在想，
怎么少一点无效上下文，
少一点重复沟通，
少一点已经做过一遍还得再来一遍的返工。

因为项目做久了以后，
Token 这东西就不只是个模型指标。

它会慢慢变成很实在的研发成本。

如果你也在长期用 AI 做项目，
真的可以抽空翻一下后台。

很多时候，
不翻一眼，
你根本不知道它到底是怎么一点点烧掉的。
```
