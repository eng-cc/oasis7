## 使用说明
- 这是第十三篇，重点不是宣布一套已经落地的治理制度，也不是泛讲“大家都是共同体”。
- 主语必须继续保持为“我 / 做游戏的人 / 我们在做的项目”，不要切成链圈治理博主、项目发币公告或商业化操盘口吻。
- 这篇的目标只有一个：
  - 讲清楚我想做的不是只有制作组在里面忙、其他人都在外面围观的游戏，而是开发者、玩家和认真把它讲出去的人都能一起参与把游戏做起来。
- 当前素材包已补齐：
  - 单图封面版：用于首屏停留和主题钩子。
  - 4 页轮播版：用于把“不是围观”“你已经在参与”“我不想做这种关系”“边界与提问”拆成更适合滑读的停顿点。
- 这篇必须保留的落点：
  - 很多游戏把玩家当流量，把开发者当执行，把传播者当外包渠道。
  - 我更认同的方向，是让开发者、玩家和认真把项目讲出去的人，都成为“共同把游戏做出来的人”。
  - “共同参与”不等于今天立刻拿到固定权益，更不是“发一篇就发币”。
  - 这里先讲的是关系：谁在一起把游戏往前推，谁就不该只被算作围观者。
  - 共同体的前提是持续参与、真实贡献和彼此信任，不是口号。
- 不要滑向以下方向：
  - `今天就已经拿到了法律意义上的所有权`
  - `现在所有相关人员都是链上投票权所有者`
  - `持币即治理`、`现在就能参与链上决策`
  - `发帖就有奖励`、`贡献了就自动发币`
  - 把共同体写成泛创业鸡汤或抽象社区口号
  - 借题发挥暗示 `oasis7` 已上线、可玩、可投资或已经开放完整治理
- 这次语气要额外做到一件事：
  - 让读者先看到“原来你说的是一种一起做游戏的关系”，而不是先看到制度词。

### 正文
```text
我越来越想做的，
不是一种只有制作组在里面忙，
其他人都在外面看的游戏。

我想做的是，
开发者在做，
玩家也在做，
认真把它讲出去的人，
也在做。

你写代码，
你提反馈，
你测 bug，
你解释它到底想做什么，
你把它带给更多人。

这些都不是路过。
这些都在把一个游戏真的做出来。

很多项目最别扭的地方是，
平时把人放在外面，
需要热度了叫玩家，
需要传播了找创作者和媒体，
需要补位了再想起开发者。

我不想做这种关系。

我更想做的，
是让真正出了力的人，
从一开始就被算作参与者。

这里说的不是
今天已经开放的法律股权、
链上投票权，
也不是自动奖励承诺。

我只是越来越想做一个，
开发者、玩家、
认真讲它的人，
可以一起参与把它做起来的游戏。

如果有这样的游戏，
你最想怎么参与？
```

## 小红书发布前自检
- 标题必须控制在 20 字内，并保持“人人都是游戏builder”这类直接把参与关系说透的表达，不再回退到抽象所有权提问句。
- 正文前半段必须先把开发者、玩家、传播者放进同一个“共同把游戏做出来”的场景里，不要一上来讲制度。
- 正文里必须同时出现“这些都在把一个游戏真的做出来”和“不是今天已经开放的法律股权、链上投票权，也不是自动奖励承诺”这两组锚点。
- 如果发轮播版，第 1 页必须先把“不是围观 / 一起做起来”立住，第 4 页必须保留边界句和唯一互动问题。
- 不出现 `法律股权已开放`、`链上投票权已开放`、`持币即治理`、`发帖就发币`、`现在就能参与治理` 这类制度性 overclaim。
- 不出现 `已上线`、`现在就能玩`、`公测`、`正式发布` 这类阶段越界表述。
- 结尾只保留一个互动问题，不追加下载、私信或投资导向 CTA。

## 评论区回复边界
- 如果有人追问“是不是已经有投票权 / 股权 / 发币安排”，统一回到一句：这里讨论的是谁在一起把游戏做出来，不是当前已经开放的权利清单。
- 如果有人质疑“玩家或传播者为什么也算”，就把讨论拉回真实贡献：谁在持续投入判断、时间、作品和信誉，谁就在参与把这个游戏做出来，不该只在项目需要的时候被看见。
- 如果有人把“共同体”理解成空话，可以直接补一句：共同体不是口号，它的前提是持续参与、真实贡献和彼此信任。
