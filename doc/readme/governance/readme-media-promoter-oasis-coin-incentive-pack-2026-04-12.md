# Media Promoter Oasis Coin Incentive Pack（2026-04-12）

审计轮次: 1

## Meta
- Owner Role: `liveops_community`
- Review Role: `producer_system_designer`
- Incentive Asset: `绿洲币 / Oasis Coin`
- Scope: `媒体、KOL、自媒体创作者、社区搬运号、普通宣传参与者`
- Rule: 宣传方也是生态参与者和受益者，但奖励基于可审计贡献，不按播放量买量

## 1. 原则
- 奖励宣传贡献，不奖励空流量。
- 奖励事实准确和新增价值，不奖励夸大承诺。
- 奖励能把讨论、协作或生态回流带回 `oasis7` 的宣传资产，不奖励纯刷屏。
- 奖励对象覆盖所有宣传参与者，但是否进入奖励建议池由贡献质量决定，不由头衔决定。

## 2. 覆盖对象
以下对象都可以进入同一套审核机制：
- 外部媒体 / 垂类媒体
- KOL / 内容创作者 / 视频作者
- 自媒体长文作者 / 串文作者 / newsletter 作者
- 社区搬运号 / 二次整理账号
- 普通宣传参与者，只要其内容有真实新增价值

## 3. 可计分与不可计分
默认可计分：
- 有独立观点、整理和判断的深度报道、测评、采访或解析
- 原创视频、长帖、串文、newsletter、播客等内容资产
- 带有二次整理、准确导读或讨论组织能力的社区扩散内容
- 带来高质量讨论、issue、PR、采访对接、合作线索或持续关注的宣传资产

默认不单独计分：
- 纯转发、纯搬运、纯截图复读
- 只有播放量、没有新增观点或事实整理
- 买量、互刷、刷评论、刷点赞
- 抄袭、拼接、断章取义
- 把 `limited playable technical preview` 说成正式上线、公测或已深度集成

## 5. 加分与扣分
### 5.1 加分
- `+15`: 证据非常完整，事实准确，能直接复核
- `+15`: 评论区或后续讨论质量高，出现具体问题、具体判断或深入追问
- `+20`: 明确带来生态回流，例如 issue、PR、采访约访、协作线索、长期 creator follow-up
- `+10`: 持续连载或系列化跟进，不是一次性发完即弃

### 5.2 扣分
- `-10`: 信息重复、增量有限或只有表层改写
- `-20`: 关键信息不准、边界含混、需要大幅人工纠偏
- `-30`: 明显 overclaim，例如写成已上线、公测、正式平台集成
- `-40`: 买量、互刷、抄袭、截图造假、多账号重复申报

## 10. FAQ
### Q1. 这是不是按播放量买量？
不是。播放量只能作为辅助信号，不能单独决定奖励。核心看内容质量、事实准确、讨论质量和生态回流。

### Q2. 普通用户发宣传内容也能进池吗？
可以。只要有真实新增价值、事实准确且可审计，不要求必须是头部媒体或 KOL。

### Q3. 发一条内容能保证拿到多少绿洲币吗？
不能。当前只做建议档位，不公开固定额度，也不做固定换算。

### Q4. 如果内容写得很热闹，但把项目状态写错了怎么办？
直接阻断奖励。越界宣传会伤害项目可信度，不能因为流量高就放过。

### Q5. 如果同一内容发了多个平台，能不能重复算？
默认只保留一个主资产 full credit，其他平台作为补充证明或小额加分，不重复 full credit。
