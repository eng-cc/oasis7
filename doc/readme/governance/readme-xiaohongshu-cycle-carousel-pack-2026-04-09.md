## 使用说明
- 这是第十二篇的轮播版，不替代长文版；它解决的是“手机端更容易读完”和“每页只记住一个判断点”。
- 标题、边界、互动问题必须和长文版保持一致：
  - 标题不变。
  - 不滑向 `哪些岗位马上没了` 这类 panic 预言。
  - 不滑向 `只要持续学习就好了` 这类空泛安慰。
  - 评论区仍然收束到“你最该主动拿 AI 去重做的是哪一部分”。
- 轮播页的节奏要求：
  - 第 1 页先把 `岗位名不会先变 / 先变的是岗位里那层工作` 立住。
  - 第 2 页把“真正该问什么”说清楚。
  - 第 3 页把主动动作拆出来，让人知道下一步怎么做。
  - 第 4 页把判断收束到“人要往更高判断层移动”和评论区自查。

## 逐页文案

### 第 1 页 / 封面
```text
AI改写的
不是岗位

是你那层工作
先变了
```

### 第 2 页 / 真问题
```text
真正该问的
不是
“我会不会失业”

而是
“我每天最常做的那部分事
到底还值不值钱”

AI先压缩的
往往不是岗位名
是岗位里那层
重复 标准 低判断的工作
```

### 第 3 页 / 主动动作
```text
别等工具来找你

更有用的做法是
主动把自己的工作拆开去试

看它替你接走哪一层
看它是在降成本
还是在改流程

很多时候不是工具不行
是你还在拿旧流程
用新工具
```

### 第 4 页 / 收束页
```text
生产力进步
本来就拦不住

所以你得尽快往上挪
往问题定义挪
往流程重构挪
往结果负责挪

如果现在就回头看你的工作
你最该主动拿AI重做的
是哪一部分
```

## 视觉方向
- 采用 `warm editorial diagnosis / work-layer ladder` 方向：像一组温暖纸面上的职业诊断卡，而不是 AI 焦虑海报或长文截图。
- 每页都应该像同一份“岗位层级观察记录”里的不同章节：停住、诊断、动作、上移。
- 重点不是“AI 很可怕”，而是“你该怎么看清岗位里正在变化的那一层工作”。
- 可用的视觉暗示：
  - 暖纸面、红棕强调、橄榄色辅助、网格线、栏线、索引标签
  - 诊断卡、步骤卡、层级台阶、上移箭头、问题标签
  - 大标题 + 短判断句，不做密密麻麻的文章截图
- 不要出现：
  - 蓝紫赛博 AI、机器人吞掉岗位、芯片脑回路
  - 四张都做成同一种封面海报
  - 过满的金句墙或鸡汤海报
