## 1. 一句话解释

`oasis7` 是一个文明模拟游戏。LLM 驱动的 Agent 会在一个持续运行的小行星带世界里采集、生产、交易、结盟、冲突；资源、制度和去中心化共识都不是背景板，而是真会限制行为的东西。

玩家不是直接点每个单位动作的人，更像世界外的战略引导者。你给出目标，调整策略，推动治理，部署模块，然后看这个世界会怎么回应，也承担这些决定带回来的后果。

所以它不是一个“给 NPC 套聊天壳”的应用，也不是一个靠静态脚本往前推的沙盒。这个项目真正想试的是：当 Agent 有长期记忆，受世界规则约束，还能通过 WASM 扩展能力时，一个世界能不能自己长出组织、市场、协议和冲突。

---

## 2. 为什么要做这个项目

很多 AI 游戏或 Agent Demo 的问题，不一定出在模型上，更常见的是世界不够结实。

常见情况是：Agent 很会说，但在世界里不承担后果；规则更像演示脚本，没有资源守恒、时间推进和可审计状态；玩家像在拉木偶线，而不是在影响一个自治系统；Demo 亮点看完之后，系统本身撑不起长期博弈。

`oasis7` 基本是反着来。它不先追问“怎样做一个更会聊天的角色”，而是先把几个硬问题摆在前面：世界时间能不能持续推进，计算和能源有没有真实成本，玩家能给到多大影响力但又不能直接夺走 Agent 的自主权，社会规则能不能从固定脚本变成可演化的模块，关键状态变化是不是能被记录、校验和回放。

这些问题不够炫，但如果不先回答，后面的文明、治理、战争、经济，最后都容易退回演示。

---

## 3. 项目主张

`oasis7` 的基本立场不复杂，主要有四点：

1. 它首先是一个游戏，而不是只展示技术栈的系统样品。
2. 它追求的是可持续运行的文明模拟，而不是一次性剧情体验。
3. 玩家的角色是战略引导者，不是世界内实体的直接操作者。
4. 社会结构、产业结构和治理结构应该尽量从世界内行为中长出来，而不是预设成一套静态剧情。

说得再直一点，项目关心的不是把 AI 展示做得更热闹，而是把世界做出来，让历史、组织和后果能在里面留下来。

这也是为什么仓库当前的公开定位已经收口为“文明模拟游戏”，同时继续保留“技术预览、尚不可玩”的边界，而不是把它包装成一个已经面向大众开放的成品。

## 4. 愿景与玩家幻想

如果只讲系统怎么跑，这个项目会像一套复杂的工程说明书。愿景这一段要回答的是另一件事：如果这个世界真的成立，玩家为什么要留下来。

我想做的，不是一个只能围观的自动系统，而是一个能给人持续参与感和成就感的世界。玩家进来以后，可以自由探索，可以试错，可以建设，也可以结盟、对抗、改规则。世界会记住这些事，后面的人也会受到影响。

这件事的重点不只是“内容更多”或者“技术更强”。更重要的是，工业、贸易、联盟、战争、治理这些东西最好不要拆成几套互不相干的玩法，而是都落在同一个持续世界里，彼此真的会发生后果。

我也希望创作者不只是消费者。后面如果条件成熟，创作者应该能发布自己的 `mod / 模块`，围绕不同规则去组织不同玩法。那时 `oasis7` 不只是一个官方版本的游戏，也会慢慢长成一个能承载不同世界秩序和社区组织方式的底盘。

产品路径会分成两个阶段：

- 第一阶段：先把 `oasis7` 做成一个开放世界沙盒游戏。重点是把参与感、探索感和成就感先做扎实，让玩家愿意在同一个持续世界里待下来，也让创作者开始能用 `mod / 模块` 组织玩法。
- 第二阶段：再把它演进成一个开放世界游戏平台。到那时，`oasis7` 不再只是一个固定玩法的世界，而是一个可以承载不同规则集、不同 mod 生态和不同参与角色的开放平台。

如果这条路走得通，玩家最后记住的应该不是某个模块名，也不是某条技术链路，而是更具体的东西：

- 我搭起来的产线，后来真的变成了贸易、联盟或者战争的基础设施。
- 我支持过的协议，改了别人之后的选择空间。
- 我花进去的时间，不是打一局就清零，而是在一个持续世界里留下了痕迹。
- 如果我愿意，我不只是在玩这个世界，也可以参与定义它以后还能怎么玩。

更长远的目标，是让游戏本身以及游戏里的关键数据，不长期锁在单一运营者手里，而是逐步走向由所有参与者共同塑造、共同治理、共同受益。这里说的是愿景和目标态，不是当前已经完成的法律结构、平台治理结构或链上实现事实。

这层愿景还是得写出来。没有它，前面那些 persistent world、world-first、real consequence 的约束看起来只是在给工程加难度；把目标说清楚之后，为什么要这样做，才比较说得通。

## 5. 核心设计原则

当前仓库里最稳定的五条设计原则是：

### 5.1 World-first

所有有效行为都必须通过世界规则验证。  
不能跳过 runtime、不能绕过共识、不能用“后台改表”替代世界内动作。

### 5.2 Emergence-first

不预设完整剧情，不强行规定每个组织、市场或制度应该长成什么样。  
项目更关心“规则和资源约束会催生什么”，而不是“作者想让玩家看到什么”。

### 5.3 Persistent

世界应当持续运行，而不是随着某个玩家离线就暂停。  
玩家离开后，世界内组织、产线、协议、冲突与后果仍然继续推进。

### 5.4 Auditable

关键状态变化要能留下事件、回执、证据和回放依据。  
没有可审计性，就无法谈真正的世界状态、真正的治理，也无法做长期运营。

### 5.5 Extensible

高阶行为不应该都硬编码在内核里。  
`oasis7` 把很多制度化、产业化、组织化能力交给 WASM 模块，让世界规则保持内核最小化、上层结构可演化。

---

## 6. 世界是什么样的

当前世界观和规则抽象有几个关键点：

- 空间背景是一个 `100km × 100km × 10km` 的破碎小行星带。
- 文明主体是硅基智能体，不围绕生物需求组织生活。
- 关键约束维度不是“吃喝睡”，而是电力、算力、存储、带宽等工程资源。
- 世界时间持续推进，不允许靠暂停来做最优解搜索。
- 资源不能凭空生成，生产、移动、计算和冲突都需要付出成本。

在实现分层上，当前仓库把资源区分为两层：

- 内建资源：由 runtime / 共识直接校验，目前以 `Electricity`、`Data` 为核心。
- 模块扩展资源：由 WASM 模块定义，用来承载更细的产业资源、信用、许可、税票、组织积分等制度化对象。

这意味着 `oasis7` 的目标不是把所有经济规则一次性写死，而是先冻结一层最小可验证世界，再允许上层制度演化。

从公开介绍的角度看，这个世界有三个最重要的气质：

- 它是活的：时间继续流动，历史会积累。
- 它是硬的：资源、移动、计算和冲突都要付成本。
- 它是可塑的：很多社会结构不是预设剧情，而是世界内主体自己造出来的。

---

## 7. 玩家、Agent 和世界的关系

这里有一条边界非常关键：玩家不是世界中的实体。

玩家更像世界外的战略提供者，可以：

- 给 Agent 设定高层目标。
- 调整提示词或决策偏好。
- 通过开发和部署 WASM 模块影响世界能力。
- 通过治理、联盟、冲突决策去改变世界结构。

玩家不能：

- 直接接管 Agent 的每一步动作。
- 绕过世界规则直接改状态。
- 绕过共识层强行写入“结果”。

这条边界决定了 `oasis7` 和传统 RTS、建造游戏、甚至很多“AI 游戏”都不一样。  
玩家要学会管理目标、约束、组织和后果，而不是用手速替代系统判断。

---

## 8. 游戏到底玩什么

从玩法目标看，`oasis7` 不是把“AI 很强”当成乐趣本身，而是把“在一个持续运行的文明里建立能力、承担风险、争夺影响力”当成乐趣来源。

它想给玩家的，不是一次性的关卡胜负，而是一个会随着选择不断改变面貌的世界：今天你只是稳住一条产线，明天你可能在谈判桌上改写联盟关系，再往后，你会发现自己已经在推动一个文明的工业、治理和战争机器。

当前玩法骨架强调三层循环：

### 8.1 微循环（5 到 15 分钟）

玩家上线后应该能看到世界变化，发现问题和机会，设定新目标，然后等待反馈。

这里的关键不是点按钮，而是判断：

- 哪条产线卡住了？
- 哪个组织关系在变化？
- 哪个风险需要先处理？
- 现在该扩张、谈判，还是暂时防守？

### 8.2 中循环（1 到 3 天）

玩家应该能完成更可见的阶段成果，例如：

- 跑通第一条稳定产线。
- 形成第一座持续运转的工厂单元。
- 推动一个组织协议通过。
- 完成一次联盟谈判或战术冲突。

### 8.3 长循环（数周）

项目真正想做的是文明级目标：

- 控制关键区域。
- 建立大型组织。
- 争夺治理权和协议解释权。
- 让不可逆历史事件在世界里留下痕迹。

换句话说，这不是一个“刷日常资源”的游戏。  
它想让玩家参与文明的形成、冲突与制度演化，并让每一轮选择都成为下一轮故事的地形。

## 9. 技术架构为什么长这样

`oasis7` 当前的高层架构大致是：

`LLM Agents -> WASM Modules -> World Runtime -> Consensus Layer -> Distributed Storage & Networking`

这条链路背后的意图很明确：

- LLM 负责决策和解释空间，不直接充当世界真值。
- WASM 负责把很多高阶逻辑变成可部署、可审计、可升级的模块。
- Runtime 负责规则验证、资源转移和状态推进。
- Consensus 负责让世界状态具备多节点可验证性，而不是单机幻觉。
- Distributed Storage / Networking 负责让状态传播、复现和长期持久化成为可能。

这种设计的代价也很明显：系统复杂度更高，工程收口更慢，很多“看起来简单”的产品动作都必须穿过更重的真值边界。

但这正是项目的核心选择。  
它宁可慢一点，也不想靠破坏世界一致性的方式去换一个短期更顺滑的 Demo。

## 10. 当前到底做到哪一步

这里最好把“能研究”“能验证”“能公开试玩”三个层次拆开看。

### 10.1 当前已经明确的部分

- 项目主定位、世界规则和玩法骨架已经有正式文档入口。
- runtime / viewer / testing / world-simulator 等模块已经形成较完整的工程文档树。
- 仓库、站点和 GitHub Releases 已经能承载技术预览阶段的构建、手册与验证说明。
- 浏览器与公开访问面的正式口径已经冻结为双模式 taxonomy：
  - `viewer`：主要正式 Web / UI 入口（旧 `software_safe` 仅保留为兼容 alias）。
  - `pure_api`：无 UI、自动化、长稳与集成入口。

### 10.2 当前还不能误读的部分

- 当前仍是技术预览阶段，尚不可玩。
- 这不是面向大众玩家的正式发布，也不是已经开放赛季运营的在线世界。
- 浏览器可验证、构建可下载，不等于“已经可以直接对外大规模试玩”。
- 很多 formal gameplay、长稳、运营放行结论，仍然需要继续靠证据链收口，而不是靠单次演示宣布完成。

### 10.3 当前最真实的公开边界

如果今天第一次接触这个项目，最合理的理解方式是：

- 你已经可以读懂它想做什么。
- 你已经可以检查它的文档、架构、验证链路和部分构建入口。
- 你已经能看到它试图建立的世界轮廓。
- 你还不应该把它当成一个已进入大众可玩阶段的成熟产品。

## 11. 这个项目和常见 AI 游戏 / 链游 / 沙盒有什么不同

如果用对比法来理解，`oasis7` 的不同点主要有这些：

### 11.1 它不是“Agent 陪聊产品”

Agent 不是为了持续扮演人格而存在，而是为了在世界里行动、承担成本、形成组织关系和长期后果。

### 11.2 它不是“区块链外壳 + 常规游戏内核”

链上部分不是装饰性的资产层，而是世界状态一致性、审计和治理边界的一部分。

### 11.3 它不是“只展示系统复杂度的工程项目”

玩法主文档已经明确：核心目标不是技术先进，而是玩家快乐、持续上线、形成社交与情绪投入。

### 11.4 它不是“作者全控的叙事体验”

世界规则给边界，具体制度和社会结构则尽量交给世界内主体演化。

### 11.5 它也不是“今天就承诺 everything everywhere all at once”

项目对公开边界是保守的。  
能确认的就确认，没完成的就继续落到 PRD、project、testing 和 evidence，不拿一句“未来会很厉害”替代当前真值。

## 12. 为什么这件事值得持续做

`oasis7` 的价值，不只在于它把几个热门词拼到一起，而在于它试图回答一个更难的问题：

当 AI 真正进入一个持续运行、规则可验证、资源有成本、社会结构可演化的世界时，游戏设计、系统设计和协作方式会发生什么变化？

如果这个问题被做出来，得到的不会只是一个更聪明的 NPC 系统，而可能是一种新的游戏结构：

- 玩家不再主要扮演操作者，而是扮演战略引导者。
- 组织和制度不再只是剧情设定，而变成游戏内真实对象。
- 模块、协议、治理、战争与工业成长会被拉进同一个世界里。
- “长期可玩性”不再只靠内容更新堆出来，而能从文明演化本身长出来。

这正是 `oasis7` 当前最值得观察的地方。它真正有潜力的地方，不是某个单点功能多新，而是这套世界如果成立，玩家讲出的故事会天然带着别的游戏很难复制的分量。
